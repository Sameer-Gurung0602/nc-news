This API allows for users to post articles and comments. It also allows for users to post comments/articles as well as vote and delete them.

Available Endpoints

Endpoint Description Response
/api The homepage of the api, provides details of available endpoints 200- OK
/api/topics Fetches all topics 200- OK - A list of all topics
/api/articles Fetches all articles 200- OK - A list of all topics
/api/users GET Fetches all users 200- OK - A list of all users
/api/articles/:article_id GET Fetches articles by id PATCH updates articles by id 200- OK - article object
/api/articles/article_id/comments GET - fetches comments by article id POST - adds comment to article 200 -OK - List of comments 201 -CREATED - updated comment
/api/comments/:comment_id DELETES comment by id 204 -no content - empty object

Example Request Body

When making a POST request to /api/articles/article_id/comments, the request body should be structured like:

Example request body: {inc_votes : 99999}

When making a PATCH request to /api/articles/:article_id, the request body should be structured like:

Example request body: {inc_votes : 99999}

Queries

On the endpoint /api/articles, the endpoint accepts the following queries:

  1. sort_by
  2. order
  3. topic

The queries can be used on their own, for example:

/api/articles?sort_by=article_id
/api/articles?order=ASC
/api/articles?topic=mitch

The queries can also be used in combination, for example:

/api/articles?sort_by=created_at&order=ASC&topic=mitch
